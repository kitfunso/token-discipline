# Contributing

Contributions are welcome.

Good contributions for this repo:
- better real-world examples
- clearer budget or stop-rule wording
- framework-specific snippet variants
- realistic eval prompts
- before/after demonstrations
- OpenClaw integration notes
- helper scripts that stay optional
- benchmark workflows that are easy to reproduce

Please keep contributions:
- practical
- portable where possible
- concise
- grounded in real failure modes
- honest about what is measured vs illustrative

If you propose a new rule, explain:
1. what failure mode it fixes
2. why it reduces waste
3. what the trade-off is

If you propose benchmark data, include:
1. the platform used
2. the prompt set used
3. what was actually measured
4. enough detail for someone else to rerun it
